# Behavioral - Week 3

## STAR Stories to Prepare

### Code Quality / Best Practices
- **Situation**: Technical debt or code quality issues
- **Task**: Improve maintainability, reduce bugs
- **Action**: Refactoring strategy, testing approach, code reviews
- **Result**: Bug reduction, faster feature development, team velocity

### Trade-offs / Decision Making
- **Situation**: Conflicting requirements or technical constraints
- **Task**: Make decision with trade-offs
- **Action**: Analysis of options, stakeholder discussions, documentation
- **Result**: Decision outcome, lessons learned, impact on project

### Learning / Growth
- **Situation**: New technology or domain knowledge needed
- **Task**: Learn and apply quickly
- **Action**: Learning approach, resources used, practice method
- **Result**: Implementation success, knowledge transfer to team

## Question Themes
- "Tell me about a time you..."
  - Refactored legacy code
  - Made a trade-off between performance and maintainability
  - Learned a new technology for a project
  - Improved code review process
  - Handled conflicting priorities

## Preparation
- Review all stories (Week 1-3): ensure variety
- Practice 2-min versions of each story
- Prepare for "What's your weakness?" (technical focus)
- Research company values, align stories

# Behavioral - Week 2

## STAR Stories to Prepare

### Distributed Systems
- **Situation**: Distributed system challenge or failure
- **Task**: Design or fix distributed system component
- **Action**: Consensus algorithms, fault tolerance, monitoring
- **Result**: System reliability, availability metrics

### Data Consistency
- **Situation**: Data consistency challenge in distributed system
- **Task**: Ensure data consistency across services
- **Action**: Transaction design, eventual consistency patterns
- **Result**: Consistency guarantees, system reliability

### System Reliability
- **Situation**: System reliability or availability issues
- **Task**: Improve system uptime and fault tolerance
- **Action**: Redundancy, monitoring, failover strategies
- **Result**: Uptime improvement, MTTR reduction

## Question Themes
- "Tell me about a time you..."
  - Worked on a distributed system
  - Handled data consistency issues
  - Improved system reliability
  - Designed for fault tolerance
  - Worked with message queues or event systems

## Preparation
- Add distributed systems stories
- Practice explaining consensus and consistency
- Prepare examples of CAP theorem trade-offs
- Research company's distributed systems challenges
